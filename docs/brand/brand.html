06 — Principles

Engineering Principles

01
Never Block the Agent
Every error path defaults to allow and continue. The hook response latency is sacred. Record second, respond first.
02
Respond First, Record Second
DB writes never sit in the critical path of a hook response. Async always. The agent keeps moving.
03
Every Write Is Idempotent
Retries, crashes, and replays produce the same state. Idempotency keys on every operation. No duplicates.
04
Type Safety End to End
Drizzle schema to Drizzle types. Zod schema to TypeScript types. No manually maintained interfaces at boundaries.
05
Mode Is a Storage Adapter
Solo and team share all service logic. The adapter pattern means if (mode === 'solo') never appears in business code.
06
Normalize at the Boundary
Each agent's payload shape is translated to HookEvent exactly once, at ingress. Everything downstream is agent-agnostic.

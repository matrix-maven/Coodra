## Decisions made

All five from the kickoff sign-off (locked in S0, see `docs/feature-packs/08a-cli/spec.md` ¬ß11):

- **Decision 1** ‚Äî `team login` / `team logout` ship as stubs in 08a. **Rationale:** locks the surface (name, flags, exit codes) so M04 doesn't churn integration; defers the OAuth body until team mode is reachable. **Alternative:** wait until M04 to even name the command. **Cross-ref:** spec ¬ß11 Decision 1 + commit `cdce23d`.
- **Decision 2** ‚Äî `~/.coodra/` honours `$XDG_CONFIG_HOME` on Linux when set; `$HOME/.coodra/` default elsewhere (including Windows). **Rationale:** Linux XDG-respect is ~5 lines (`env-paths`); Windows `%APPDATA%` integration is genuinely non-trivial, defer if user demand surfaces. **Alternative:** hand-rolled `path.join(homedir(), '.coodra')` everywhere. **Cross-ref:** `src/lib/coodra-home.ts`.
- **Decision 3** ‚Äî `init` re-run is idempotent merge by default; `--force` is the destructive override. **Rationale:** matches M02 verification F13 closure mindset; CI / scripted setups stay safe. **Alternative:** prompt-and-skip-per-file (would re-add the `prompts` dep and break `--json`/non-TTY). **Cross-ref:** `src/lib/init/{coodra-json,env-merge,mcp-merge}.ts` with `WriteOutcome.action: 'wrote' | 'merged' | 'unchanged' | 'forced'`.
- **Decision 4** ‚Äî `doctor` is strictly read-only in 08a; no `--fix` mode. **Rationale:** doubles the test matrix and introduces silent-recovery footguns; every red maps to `init` or a documented one-liner. **Alternative:** `--fix --apply` flag pair. **Cross-ref:** `src/doctor/`.
- **Decision 5** ‚Äî Standalone npm package `@coodra/cli`; `package.json` ships with `bin` + `files` + `engines` + `repository` + `publishConfig` from S1. Publication step out of 08a scope. **Rationale:** locks the tarball shape today so publish-flag-day is a one-liner. **Alternative:** workspace-only bin. **Cross-ref:** S9 npm-pack file-list lock test.

Three implementation decisions made during the slice work (not blocking, but non-obvious):

- **Per-check timeout in doctor runner.** Each check runs against `Promise.race(check.run(), timeoutPromise(timeoutMs))`. A check that times out is counted as `fail` in the summary ‚Äî a check too slow to certify the invariant is not certifying anything. **Cross-ref:** `src/doctor/run.ts` + the slow-check test.
- **`openLocalDb` narrows DbHandle ‚Üí SqliteHandle inline.** `createDb({ kind: 'local' })` returns the union; downstream code needs SqliteHandle to call `handle.raw.prepare`. The wrapper makes this explicit and accepts an optional `loadVecExtension` flag (default false for read-only doctor checks; true when applying migrations). **Cross-ref:** `src/lib/open-local-db.ts`.
- **DaemonUnit + service registry.** The cli's `start`/`stop` build a `DaemonUnit` per service from `SERVICES`, hand it to the platform-selected manager. The manager owns the install / start / stop / status primitives; the cli composes them. This matches the "tool descriptions as agent prompts" pattern from ¬ß16 ‚Äî the abstraction lives at the right boundary. **Cross-ref:** `src/lib/services.ts` + `src/lib/daemon/`.

### Whole-product verdict

`install ‚Üí init ‚Üí start ‚Üí agent runs ‚Üí audit ‚Üí stop` works end-to-end as one shipped product after the 6 fixes. The CLI hosts both daemons; the bridge audits to the same DB the doctor reads from; the F-finding invariants (F7/F8/F9/F10/F14/F15) all hold; SessionStart/Pre/Post/Stop produce a coherent runs‚Üîrun_events‚Üîpolicy_decisions‚Üîdecisions chain.

Where it would break first under load / interrupt / a real Clerk tenant:
- **SIGTERM during a queued setImmediate audit write** ‚Äî the M03.1 known limitation (the durable-outbox feature pack at `docs/feature-packs/03.1-durable-outbox/` exists for exactly this; doctor check 13 reports it permanent-yellow until M03.1 lands).
- **Real Clerk tenant** ‚Äî solo-bypass requires `CLERK_SECRET_KEY=sk_test_replace_me`. With a real Clerk JWT, the auth chain in `apps/hooks-bridge/src/lib/auth-middleware.ts` runs `verifyClerkJwt` against the Clerk JWKS ‚Äî currently exercised only by unit tests with mocked tokens; Module 04 (Web App) is the first slice that lands a real Clerk tenant in the loop.
- **Concurrent CLIs racing on the same DB** ‚Äî better-sqlite3 + WAL handles concurrency for inserts, but two `coodra init` calls in parallel against a brand-new DB could both try to create the projects row (the `ensureProject` re-select after `ON CONFLICT DO NOTHING` handles the slug-unique race; covered by the new test).

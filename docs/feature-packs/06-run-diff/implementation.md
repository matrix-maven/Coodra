`captureBaseSha` spawns `git rev-parse HEAD` via `execFile` (no shell), validates the SHA shape (40 or 64 hex chars), then `UPDATE runs SET base_sha = ? WHERE project_id = ? AND session_id = ? AND base_sha IS NULL` with up to 3 retries × 50ms backoff for the SessionStart-vs-runs-row race. The capture is fire-and-forget from the SessionStart handler so the response goes back to the agent immediately.

## Slice 2 — SessionEnd run-diff runner

Files:
- `apps/hooks-bridge/src/lib/run-diff-runner.ts` (new)
- `apps/hooks-bridge/src/handlers/session-end.ts` — `scheduleAutoContextPackSave` now awaits `runRunDiff` before `saveAutoContextPack`
- `apps/hooks-bridge/__tests__/unit/lib/run-diff-runner.test.ts` (new) — 6 tests

Runner flow:
1. `loadRunMeta` — SELECTs `runs.base_sha` and the run's Edit/Write/MultiEdit/NotebookEdit `tool_input.file_path`s.
2. If `base_sha IS NULL` → persist soft-failure row, return.
3. If touched-files set is empty → persist `no_edits_in_run` row, return.
4. Run 4 parallel git subprocesses: `git diff`, `git diff --numstat`, `git diff --name-status -z`, `git status --porcelain=v1 -z`. Results join into the `files_changed` array.
5. For untracked agent-touched files, read content + synthesize `/dev/null → b/path` diffs.
6. Concatenate tracked + untracked diffs, truncate at `MAX_UNIFIED_DIFF_BYTES` (256 KiB) at a clean newline.
7. DELETE existing row by `run_id`, INSERT new row.

All four git subprocesses use `--no-color`, `--no-ext-diff`, `-M -C` (rename + copy detection). Timeout 15s. Buffer caps prevent runaway memory use.
